# Retail Retention MVP

Проект для кейса по аналитике, прогнозу риска ухода клиентов и персональным рекомендациям для удержания.

## Что внутри

- `app/dashboard_app.py` — интерактивный дашборд на Streamlit
- `app/recommender_engine.py` — гибридный рекомендательный движок
- `notebooks/eda_starter.ipynb` — стартовый анализ данных
- `notebooks/customer_table_prep.ipynb` — подготовка клиентской таблицы
- `notebooks/churn_target_prep.ipynb` — формирование целевого признака ухода
- `notebooks/churn_model_baseline.ipynb` — базовая модель ухода
- `notebooks/churn_model_advanced.ipynb` — усиленная модель ухода
- `notebooks/retention_recommendations.ipynb` — логика удерживающих рекомендаций
- `notebooks/recommender_hybrid_engine.ipynb` — исследование рекомендательного движка


## Как запустить дашборд

1. Создать окружение и установить зависимости:

```bash
pip install -r requirements.txt
```

2. Положить файлы `data.csv` и `events.csv` в папку `data/`

3. Запустить приложение:

```bash
streamlit run app/dashboard_app.py
```

## Как устроено решение

1. Выполняется исследовательский анализ данных и очистка.
2. Формируется клиентская витрина с признаками по заказам и событиям.
3. Для клиентов считается риск ухода.
4. Для выбранного клиента автоматически подбирается режим рекомендаций:
   - обычный,
   - удерживающий,
   - для лояльных клиентов.
5. Гибридный рекомендательный движок показывает товары и объясняет, почему они попали в выдачу.
